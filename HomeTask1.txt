 1. Как скопировать на свой компьютер репозиторий из github.com,
 что должно быть сделано предварительно?



git config --global -l  система покажет какие настройки есть в нашем GIT
git config --global user.name MaratGafiev

git config --global user.email gmkorg109@gmail.com  добавление имя почты, если их нет. Это информация о том, кто добавляет commit


На компьютере должна быть рабочая директория, куда вставляется ссылка на новый созданный репозиторий в GitHub (включая выбор файла README).

git clone git@github.com:maratgmk/MaratGafiev.git вставляем ссылку созданного репозитория GitHub в свою рабочую директорию на Git Bash

перейдем в папку cd MaratGafiev    =>   ls -al и убедимся  что директория репозитория Git существует на компьютере

git remote -v     псевдоним репозитория    origin (чтобы не набирать название репозитория)
(fetch) откуда будем получать == извлекать, вызывать, приносить убитую дичь
(push) куда будем посылать

2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?



Если создать папки и файлы на компе, потом командой git init создаем новый репозиторий на GitHub 
git init обратный путь создания нового репозитория на Git

git remote --help все подсказки

git status увидим подсказку 

git add .  добавляются все файлы новые и измененные 

git commit -m "Init commit"   все закоммитим, заполним свой репозиторий
git status
git log   локально можно работать
git remote -v   ничего не покажет, 
идем на Git создаем новый репозиторий, но галочку на README не ставим, нужен пустой репозиторий
следуем подскасзкам Git копируем ссылку "...or push an existing repository from the command line" первую строку

вставляем в Git Bash и проверяем git remote -v
должны появиться origin, fetch, push
вторая строка из подсказок Git Hub " git branch -M main" переименут branch Master на main, если это старая версия
третья строка "git push -u origin main" этот вариант делает два в одном, и файлы пушит и переименовывает branch

3. Как получить на локальный компьютер изменения с github.com?

комманда "pull"

4. Какую информацию нам дают команда :
git status
git log
git remote -v 

набираем команду git status (untracked files) файл красный
git add song.txt
git status файл позеленел и перешел в индексовую директорию staged
git commit -m "Add new file song.txt"     файл добавили в "нулевой" версии, закомментировали
git status еще раз и наблюдаем все изменения

git log выводится информация о всех коммитах, кто их сделал и когда , и какой секретный код в 16-ричной системе счисления


git remote -v показывает удаленую связь и названия репозитория на GitHub
псевдоним репозитория origin (чтобы не набирать название репозитория)
(fetch) откуда будем получать == извлекать, вызывать ==  pull тянуть
(push) куда будем посылать
